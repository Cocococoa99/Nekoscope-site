Privacy Policy

Last updated: May 29, 2026

NekoScope is a read-only creator analytics tool. This Privacy Policy explains what information NekoScope processes when a creator authorizes TikTok Login Kit and uses the local analytics application.

Information we process

  • TikTok account information authorized by the user, such as open_id, display name, avatar URL, profile link, bio, verification status, follower count, following count, likes count, and video count.
  • Public video information authorized by the user, such as video id, creation time, title or description, duration, cover image URL, share URL, view count, like count, comment count, and share count.
  • Local analytics records entered or imported by the user, such as manual notes, CSV or Excel imports, screenshots, and metric snapshots.
  • OAuth access tokens and refresh tokens needed to access TikTok APIs after the user grants authorization.

How information is used

Information is used only to provide creator analytics features, including importing authorized TikTok video data, reconciling API data with local records, generating local performance snapshots, and helping the creator review their own content performance.

Local storage

NekoScope is designed as a local Streamlit application. OAuth tokens and analytics data are stored in the user's local SQLite database on their own device. This static website does not store OAuth tokens or user analytics data.

No posting or automation

NekoScope does not use the Content Posting API. It does not upload, publish, edit, delete, like, comment, follow, message, scrape other users, or perform automated TikTok account actions.

Sharing

NekoScope does not sell user data. User data is not shared with advertisers or data brokers. Data is only processed to provide the local analytics functions requested by the authorized user.

User control and deletion

Users can revoke TikTok authorization in TikTok settings or clear local tokens inside NekoScope. Users can also delete the local SQLite database or exported files from their own device.
